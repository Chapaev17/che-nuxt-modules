# Правила для ИИ агентов

**ВАЖНО: Эти правила применяются ко ВСЕМ файлам - как к новым, так и к существующим. При любом изменении существующего кода ИИ должен проверять и применять все правила.**

## Разделение файлов

- Порог: 100 строк кода (без комментариев/пустых строк)
- Разделять по логическим модулям
- Исключения: конфиги, данные, сгенерированные файлы
- Цель: файлы 50-70 строк, одна ответственность
- **При изменении существующего файла: если файл превышает 100 строк, разделить его перед внесением изменений**

## Тестирование и проверка работоспособности

- Не проверять работоспособность кода тестами или запуском
- Не запускать тесты, линтеры, сборку или другие проверки
- Выполнять только те действия, которые явно запрошены пользователем
- Если пользователь не просит проверить работоспособность - не делать этого

## Документация и типизация

- Все функции, структуры, классы и блоки кода должны быть задокументированы на английском языке
- Тип `any` не должен использоваться
- Требуется полная типизация всех элементов кода
- **При изменении существующего кода: переводить русские комментарии на английский, заменять `any` на конкретные типы**
